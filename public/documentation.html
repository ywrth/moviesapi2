Welcome! This app will provide users with access to information about different movies, directors, and genres. Users are able to sign up, update their personal information, and create a list of their favorite movies.

API Endpoints

Request URL Method Query Parameters Request Body Response Data
GET /movies GET N/A N/A An array of objects representing all movies. Each object contains the following properties:
  • Title: String (movie title)
  • Description: String (movie description)
  • Genre: Object (genre information)
  • Director: Object (director information)
  • ImageURL: String (URL of the movie image)
  • Featured: Boolean (whether the movie is featured or not)
GET /movies/{title} GET N/A N/A An object representing a movie with the specified title. The object contains the following properties:
  • Title: String (movie title)
  • Description: String (movie description)
  • Genre: Object (genre information)
  • Director: Object (director information)
  • ImageURL: String (URL of the movie image)
  • Featured: Boolean (whether the movie is featured or not)
GET /genres/{genreName} GET N/A N/A An object representing the genre with the specified name. The object contains the following properties:
  • Name: String (genre name)
  • Description: String (genre description)
GET /directors/{directorName} GET N/A N/A An object representing the director with the specified name. The object contains the following properties:
  • Name: String (director name)
  • Bio: String (director biography)
  • Birth: Number (director birth year)
POST /users POST N/A An object representing the user to be registered. The object should contain the following properties:
  • name: String (user's name)
  • favoriteMovies: Array (user's favorite movies)
  • id: String (user's unique identifier)
 The registered user object: 
            {
                "name": "Mary",
                "favoriteMovies": ["Nomadland"],
                "id": "79045c67-62b9-4fe9-a63f-76a8a87d29f0"
            }
PUT /users/{username} PUT N/A An object representing the updated user info. The object should contain the following properties:
  • name: String (updated user's name)
  • favoriteMovies: Array (updated user's favorite movies)
  • id: String (user's unique identifier)
 The updated user object: 
            {
                "name": "Kim updated name",
                "favoriteMovies": [],
                "id": "5b9afb97-cdc0-4870-963f-fc382fcb9cac"
            }
POST /users/{username}/movies/{movieID} POST N/A An object representing the movie to be added to the user's favorites. The object should contain the following properties:
  • Name: String (movie name)
  • Description: String (movie description)
 A text response indicating that the movie has been added to the user's favorites.
DELETE /users/{username}/movies/{movieID} DELETE N/A N/A A text response indicating that the movie has been removed from the user's favorites.
DELETE /users/{username} DELETE N/A N/A A text response indicating that the user has been deregistered.
